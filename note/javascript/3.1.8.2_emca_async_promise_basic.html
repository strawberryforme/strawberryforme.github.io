    1、 Promise是什么。

    # 1、 Promise是什么。

    Promise和Object，Function，Array，String一样。都是构造函数。

    既然是构造函数，我们就将3部分。1. 构造函数。 2.函数方法 3.原型方法。

    # 2.new Promise

    new Promise 我们讲2个。1.传入的参数。 2返回的值。

    1.传入的参数

    //1.参数：
//1.1参数必须是一个函数。
const p = new Promise(function(){}) // ok
const p = new Promise(()=>{})//ok
//1.2参数如果不是函数就报错
const p = new Promise(not_function_excutor) // TypeError: Promise resolver undefined is not a function


//2.参数内部有函数，函数内部有以下几种情况
//2.1传入两个参数
const p =new Promise(function (res,rej){ })
//如果是同步：
//res，rej只能按照执行1个。执行1个后，res，rej在执行。promise状态不会改变。所有就没有用。

//此时就要看res先执行，还是rej先执行。
//如果 res在前面，或者settime先执行，或者if判断先执行。那么后面的rej，res则不执行。

//2.2传入1个参数
const p = new Promise(function (res) { })

//2.3不传入参数，传入同步代码,或者不传入。
const p = new Promise( function () { })

//2.4不传入参数，throw抛出错误
const p = new Promise(function (){ throw 'xxx' })
const p = new Promise(function (res){ throw 'xxx',res(1)})//res不会执行。

//2.5如果提前return 则会终止。
const p = new Promise(()=>{return 2})

//2.6 如果有new Promise。
//new Promise会立刻执行，但是回调函数会放在微任务里面。
//会根据本身的res在newPromise前面，还是后面。谁在前面，谁的微任务就在前面。
const p = new Promise(()=>{ new Promise(()=>{console.log(2)})})

//2.7 如果有宏任务。
// SetTimeout下一个任务里面。
const p = new Promise(()=>{ setTimeout(()=>{},2000) })

    2.返回值。

    //1.没有参数，参数不是函数。
const  p = new Promise()//如果没有参数。
const  p = new Promise(console.log(2))//或者参数不是函数。
//直接报错。promise内部传入必须是个函数。

//2.如果有参数，参数为函数。
const  p = new Promise(()=>{}) //函数内部没有代码。
const  p = new Promise(()=>{console.log(1)})//函数内部只有同步代码。
const  p = new Promise(()=>{return 2 }) //如果函数内部在 res,rej之前return了。
const  p = new Promise((res)=>{setTimeout(()=>{res()},2000)})//如果函数是异步返回res，或者rej。状态先为pending，后是res，rej。
// p的：
// PromiseState: pending
// PromiseResult:undefined。

//3.如果有参数，参数为函数。函数内部先执行res。
const p = new Promise((res)=>{res()})
const p = new Promise((res,rej)=>{res()})
const p = new Promise( (res,rej)=>{ setTimeout(()=>{res(1)},2000),setTimeout(()=>{rej(2)},3000) })
//记住：按照事件循环机制执行异步，但是回调函数时间 是按照时间来的。
//p的属性
//PromiseState:  fulfilled。
//PromiseResult: 'value' //value 有可能为空。undefined。


//4.如果有参数，参数为函数。函数内部先执行rej。 或者直接抛出throw异常。
const p = new Promise((res,rej)=>{rej()})
const p = new Promise((res,rej)=>{ setTimeout(()=>{res(1)},10000),setTimeout(()=>{rej(2)},3000) })
const p = new Promise( throw(3))
const p = new Promise( (res,rej)=>{throw(3),rej(2)})
//p的属性
//PromiseState:  rejected。
//PromiseResult: 'value' //value 有可能为空。undefined。

    # 3.返回的实例对象方法

    1.then

    then参数

    //1.参数为非函数：  
const p = new Promsie((res)=>{res(1)})
p.then() //参数为空
p.then(2) //参数为表达式(2)
p.then(console.log(2)) //参数为表达式console.log(1)。
p.then(Promise.resolve(2)) // 参数为表达式Promise.resolve(2)
//会造成promise击穿：返回值都是promise。pending：fulfilled。value：1


//2.参数为函数。。。函数里面没有参数，函数里面
const p = new Promsie((res,rej)=>{res(1)})
p.then(()=>{})
p.then(()=>{console.log(2)})
p.then(()=>{new Promise(2)})

//3.参数为函数。。。函数里面没有1个参数
const p = new Promsie((res,rej)=>{res(1)})
p.then((res)=>{console.log(res)})

//4.参数为函数。。。函数里面有两个参数
const p = new Promsie((res,rej)=>{res(1)})
p.then((res)=>{console.log(res)},(reason)=>{console.log(reason)})

//5.参数为函数。函数里面有return。
const p = new Promsie((res,rej)=>{res(1)})
p.then((res)=>{return 2},(reason)=>{return 3})
p.then((res)=>{return new Promise((rej)=>{rej()})},(reason)=>{return 3})

//6.参数为函数。函数里面有throw。
const p = new Promsie((res,rej)=>{res(1)})
p.then((res)=>{throw 3},(reason)=>{return 3})
p.then((res)=>{()=>{},(reason)=>{throw 4}} )

//1.1参数为非函数：
// p.then(),p.then(2),p.then(console.log(2))
//1.2忽略某个状态的回调函数参数。就是状态是rejected，但参数只有1个函数。
// Promise.reject(2).then((reason)=>{console.log(resson)})


//2.p.then返回值 =undefined。状态为fulfilled
//2.1没有返回任何值。
//p.then(()=>{},()=>{})
//p.then(()=>{ console.log(2) },()=>{console.log(2) } )
//p.then(()=>{ new Promise(2) },()=>{ new Promise(2) } )
//2.2返回了值。
//p.then(()=>{return 2})
//p.then(()=>{return 3})
//2.3返回异步值。
//p.then(()=>{ return new Promise((res,rej)=>{res(2)})},()=>{ return new Promise((res,rej)=>{res(2)})})

    2.catch

    const p = new Promise((res,rej)=>{rej(1)})
p.catch((e)=>{console.log(e)});
//p必须是 rejected。

//1.传参和返回值
//1.什么都不穿。p.catch()。 捕获异常。返回的值就是p。

//2.只传1个函数。函数里面是个空函数。返回普通值。fulfilling。
//2.只传1个函数。函数里面没有return语句。只有同步代码。 fulfilling

//4.只传1个函数。 函数里面有return语句。return返回普通值。fulfilling
//5.只传1个函数。 函数里面有return语句。return返回异步值。fulfilling。

//6.只传1个函数。 函数里面有return语句。return返回异步值reject。

//7.只传1个函数。 函数里面有return语句。return返回异步值pending。

//2.特殊情况
//如果是异步返回的是不会执行的。
var p2 = new Promise((res,rej)=>{setTimeout(()=>{throw '22'},2000)}) 
p2.catch((e)=>{console.log(e)})// 异步的操作，不会执行

    3.finally

    1.无论promise结果是多少，我们想在最终做一个处理。 finally 的回调函数中不接收任何参数，无论结果是什么都会处理。 返回一个Promise。

    const p =  new Promise(()=>{})
    p.then(function(json) { /* process your JSON further */ })
    .catch(function(error) { console.log(error); })
    .finally(function() { isLoading = false; });

    # 4. Promise方法

    1.简化我们的参数。 reslove，reject

    //一、resolve
//1.非promise的参数。 数字，数组，函数。 
//2.promise参数。 Promise.resolve(1)。new Promise((res,rej)=>{res(),rej()})

//resolve 返回值：
// 返回一个带着给定值解析过的 Promise 对象.(数字，数组，函数)
// 如果参数本身就是一个 Promise 对象，则直接返回这个 Promise 对象。

const p = new Promise((res,rej)=>{res(1),rej(2)})
const p = Promise.resolve() // promise fulfilling undefined
const p = Promise.resolve(1) // promise fulfilling 1
const p = Promise.resolve("123") // promise fulfilling "123"
const p = Promise.resolve([1,2,3]) // promise fulfilling [1,2,3]
const p = Promise.resolve(Promise.resolve(66)) // promise fulfilling 66

const qwer = Promise.resolve(Promise.reject(2)) //promise rejected 2

const qwer = Promise.resolve(throw 3)//报错。

//二、reject 
//reject 参数。 表示Promise被拒绝的原因。
//rejected 返回值。一个给定原因了的被拒绝的 Promise。
const p = new Promise((res,rej)=>{rej(2),res(1)})
const p = Promise.reject() // promise rejected undefined
const p = Promise.reject(1) // promise rejected 1
const p = Promise.reject("123") // promise rejected "123"
const p = Promise.reject([1,2,3]) // promise rejected [1,2,3]
const p = Promise.reject(Promise.resolve(66)) // promise rejected  Promise
    1. allSettled, race, all, any(捕获所有返回值，第一个完成的返回值，第一个失败的返回值，第一个成功的)
    //2.1 allSettled（适合多个彼此不依赖的异步任务完成，或者得到每个promise的结果时）
//获取容器所有的value。

//2.2 race (方法返回一个 promise，一旦迭代器中的某个 promise 解决或拒绝，返回的 promise 就会解决或拒绝。)
//我只获取其中的一个。最先完成的就行。

//2.3 all（适合相互依赖，或者其中任何一个reject时立即结束。）
// 迭起器内部都是fuilling参数，那么返回成功状态，value是所有值数组。
// 迭起器内部如果有rejected，返回最先的rejected，value是reject的value。

//参数和返回值
Promise.all();  //报错

Promise.all("")
Promise.all("123456");//参数会调用Promise.resolve，将参数转Promise实例，状态都是fulfilling。

Promise.all([]) //如果参数都是fulfilled。返回Promise，fulfilled，Array(0)。

Promise.all([12,32,"213"]) //参数没有异步的，先返回pending。随后返回fulfiled。
Promise.all([42,'23',Promise.resolve(3)]);//先返回pending。随后返回fulfiled。如果参数都是fulfilled。返回Promise，fulfilled，Array(N)。
Promise.all([42,'23',Promise.resolve(3),Promise.reject(3)]);//先返回pending。随后返回fulfiled。如果参数里面有一个rejected。返回Promise，rejected，555返回值。
Promise.all([Promise.reject(3),Promise.reject(10)]);//先返回pending。随后返回fulfiled。如果参数都rejected，会按照最先的时间进行返回rejected。返回Promise，rejected，555返回值。

//这个可以证明new Promise是同步的。Promise,then，all是异步的。
const b = new Promise((res)=>{res()});console.log(b) //fulfilled

const p = Promise.resolve(2).then(()=>{});console.log(p)//pending

const t = Promise.all([123]);console.log(t) //pendding

//2.4 any(和all相反)
//有fulfiled，有fulfiled
//否则返回所有有rejected
