    # Part2、定位

    static。的元素不会以任何特殊方式定位；它始终根据页面的正常流进行定位：。
relative：
不会失去文档中位置。
相对定位元素的定位是相对它自己的正常位置的定位。
相对于最近的父元素。根元素。
相对定位是相对于元素在文档中的初始位置；因此，移动元素会导致它覆盖其它框。

absolute：
导致元素脱离文档流.绝对定位的元素相对于 static 定位以外的第一个父元素进行定位。(父元素具有position:relative/absolute/fixed时)
如果元素没有已定位的父元素，那么它的位置相对于<html>。

绝对定位与文档流无关，所以它们可以覆盖页面上其他的元素，可以通过z-index属性来控制这些层的对方顺序
失去文档中位置。会层叠在文本流之上。
解决方法1：在js中设置父元素高度等于子元素的高度。
解决方法2：给父元素强行设置高度。（对于宽度导致的类似问题就强行设置宽度）

fixed ；的元素是相对于视口定位的，这意味着即使滚动页面，它也始终位于同一位置
sticky：的元素根据用户的滚动位置进行定位。

z-index 的默认值为 auto 。auto 。number inherit。

z-index起作用：堆叠上下文只有在postion:relative/absolute/fixed/sticky脱离文档流控制时才生效，static时无效。

z-index不起作用的几种情形
（1）当前设置z-index元素的父元素position:relative/absolute;
(2) 当前设置z-index元素为浮动元素。


比较两个 DOM 元素显示顺序 ？接下来就来总结一下如何比较两个 DOM 元素的显示顺序呢？
如果是在相同的层叠上下文，按照层叠水平的规则来显示元素
后来居上原则
谁 z-index 大，谁在上的准则
如果z-index相等。
顺序：层叠上下文。负index。 block块水平盒子。float浮动盒子。inline&inlineblock水平盒子。 auto或者为0。正的index。
父子继承问题：如果父元素没有处于堆叠上下文时，即z-index:auto;或者position:static;时，子元素不会继承父元素的优先级。
当父元素和子元素都处于堆叠上下文时，子元素继承父元素的优先级，故父元素大的就大
如果是在不同的层叠上下文中，先找到共同的祖先层叠上下文，然后比较共同层叠上下文下这个两个元素所在的局部层叠上下文的层叠水平。

使用准则：
不犯二准则：对于非浮层元素，避免设置 z-index 值，z-index 值没有任何道理需要超过 2
对于浮层元素，可以通过 JS 获取 body 下子元素的最大 z-index 值，然后在此基础上加 1 作为浮层元素的 z-index 值

    # Part3、小盒子在大盒子飘起来

    浮动的值：float：left。right 。none。inherit。
浮动原理: 块级元素脱离文档流。浮动的框可以向左或向右移动，直到它的外边缘碰到包含框 或 另一个浮动框的边框为止。
浮动特性：脱标。浮动元素之间互相贴靠。产生”字围“效果。收缩效果(当浮动元素没有设置尺寸,会适应浮动元素内的子元素尺寸)。浮动元素设置margin_top不会发生塌陷现象
浮动应用：主要应用于多个 块级元素 横向布局.让多个块级标签在一行。等宽的框。导航菜单。(块转行内块也可以实现一行显示,不过存在空白折叠现象)
浮动影响：浮动只会对文档流一行和后面的盒子产生影响。对文档流的盒子是不会影响的。
浮动变化：
    1、内联元素 在浮动时将变为块级元素。
这是因为浮动时的所有元素都将获得block其display属性（inline-table将获得table）的值，从而使它们成为块级元素。
2、未指定宽度的块元素将在浮动时自适应其内容。
通常，如果未指定块元素的宽度，则其宽度为默认值100％。但是，当浮动时，情况就不再如此; 块元素的框将缩小，直到其内容保持可见
3、浮动的父元素：
父元素不关心他们的“漂浮”子元素，除了他们不应该离开他们的左边界或右边界。
通常，未指定高度的块元素会增加其高度以容纳其子元素，但“Float”子元素不是这种情况。如果“浮动”大小增加，其父级将不会相应地增加其高度。

特殊浮动：当带有字的元素碰到浮动时会自动绕开

浮动塌陷：解决没设置高度父盒子因为子盒子浮动而造成的塌陷问题。
清除浮动:
1、内墙法：给后面的父盒子设置clear:both属性（缺陷:上面的盒子height还是0 所以margin会失效）
2、给父盒子添加用overflow:hidden。
3、使用::after伪元素给盒子添加最后一个子元素 并且给::after设置clear:both(伪元素为行内元素 务必要设置成块元素)
4、双伪元素清除浮动

浮动遇上定位：
float 可以和 relative 混用： 两个同时起作用
float 和 absolute 不可以混用。绝对或固定定位的元素将不会浮动。float不生效.一旦混用浮动和绝对定位，则绝对定位会使浮动属性失效。
一个块float和另一个absolute块的z轴位置。absolute 不设置z-index的情况下。absolute 设置z-index = -1的情况下。

4、Q:那之前的布局有哪些缺陷？Flex又是如何解决呢？    
    A:  margin，padding: 需要计算px。
        定位原理： 通过计算px进行布局。top,left等。
        定位问题: 单个元素需要计算px，考虑父元素的关系。多个元素之间关系。
        浮动原理： 通过脱离标准流进行布局。
        浮动问题：在于清除浮动。
        Flex解决：和数学一样引入轴的概念。通过对容器进行主轴、侧轴的 排序，换行，主轴定位和侧轴定位 来达到目的。
        Flex概念：2009年w3c发布一项技术。目前所有浏览器都支持。为什么是弹性。主轴，侧轴可以伸缩。来到底布局。
        Grid解决：更厉害。二维布局,将容器划分成了“行”和“列”，产生了一个个的网格。

    # 2、flex属性

    6、flex容器属性。
    学习一下英语：方向：dirtecion。修正：justifty。align对齐。
    1、flex-dirtection
        *作用:定义了主轴的方向(正方向或反方向)。
        *值:
        - 默认：row。
        - row: 设置主轴是水平方向 默认
        - row-reverse: 与 row 的方向相反
        - column：设置主轴的方向是垂直方向
        - column-reverse: 与column 方向相反
    2、flex-wrap: 
        *作用：设置伸缩容器的子元素是单行显示还是多行显示。 
        *值：
        -默认：nowrap。
        -nowrap：单行显示。挤在一起。
        -wrap：多行显示。分开。
    3、justify-content:
        * 作用：主轴的子元素排列方式(头部尾部居中，周围平分，贴边平分).    
        * 值
        - 默认值：flex-start
        - flex-start: 默认从头部开始。
        - flex-end: 从尾部开始。
        - center: 主轴居中对齐。(可以垂直，也可以水平，根据主轴来定)
        - space-around: 平分剩余空间。（剩余空间给margin-left，margin-right）
        - space-between:  先两边贴边，在平分剩余空间。（重要）
        - 没有stretch。
    4、align-items:(子项目不设置宽高)
        * 作用：侧轴的子元素排列方式。在子项目是单行的时候用。
        * 值
        - 默认值:str etch。
        - str etch: 拉伸 flex 项目以填充容器（默认）(但是不要给元素高度，否则不起作用)
        - flex-start:  从上倒下。
        - flex-end: 从下到上。
        - center: 挤在一起。
        - baseline 值使 flex 项目基线对齐
        - 没有space-around,space-between。
    5、align-content：设置子项侧轴排列方式且子项出现换行的情况(所有子项目宽度的和>父项目宽度，并且设置换行)，单行是没有效果。
        * 作用：侧轴的子元素排列方式。在子项目是单行的时候用。
        * 值
        - 默认值：stretch。
        - flex-start:  侧轴开头排列。
        - flex-end: 侧轴尾部排列。
        - center: 侧轴居中对齐。(可以垂直，也可以水平，根据主轴来定)
        - space-around: 平分剩余空间。（剩余空间给margin-left，margin-right）
        - space-between:  先两边贴边，在平分剩余空间。（重要）
        -  stretch: 拉伸弹性线以占据剩余空间（默认）。（不要给高度。）

7、flex子项属性。
    1、flex子项目占剩余空间的份数及计算。
        1、flex
            1、单值 
                1、无单位数字：
                    flex-grow:1=flex(1 1 0%)用于等分布局)
                    flex-grow:0=flex(0 1 0%)用于等分布局)
                    元素尺寸不会弹性增大（flex-grow:0）,但是会弹性收缩(flex-shrink:1），
                    考虑到此时flex-basis属性值是0%，表示基础尺寸是0，因此设置flex:0的元素尺寸表现为最小内容宽度，也就是文字会呈现“一柱擎天”的效果。
                2、有效宽度px：
                    flex: 12px = flex(1 1 12px)
                3、flex:none = flex(0 0 auto)
                    flex子项没有弹性，此时flex-basis属性值是auto，即基础尺寸由内容决定，
                    因此设置flex:none的元素最终尺寸通常表现为最大内容宽度。
                4、flex:auto = flex(1 1 auto)
                    flex:auto等同于设置flex:1 1 auto，元素尺寸可以弹性增大，也可以弹性减小，在容器尺寸不足时会优先最大化内容尺寸。
                5、flex: initial = 【默认值】flex( 0 1 auto) 
            2、双值
                1、无单位数+无单位数：flex-grow,flex-shrink
                2、无单位数+有效宽度值：flex-grow,flex-shrink.
            3、3值
                1、无单位数+无单位数+有效宽度值。flex-basis。
        2、flex-grow
            *作用：计算剩余空间。进行平分。
            *值:
                -默认值：0
        3、flex-shrink
            *作用：会缩小换行。子元素超出的宽度 * flex-shrink的值 * 子元素宽度 / 总值;最大最小尺寸限制>弹性收缩，弹性扩张>基础尺寸
            *值:
            -默认值：1
        4、flex-basis
            *作用：
            *值:
            -默认值：auto
    2、align-self
        *作用：自己在侧轴的排列方式。
        *值：
        -默认值：auto(该值使该属性直接继承其父元素的行为)
        -flex-end
        -flex-start
        -center
        -str etch
    3、order:
        *作用：子项的排列顺序。越小越靠前。

    # Part7、BFC原理

    BFC：

    页面上的一个隔离的独立容器。一个不会影响到外面的独立封闭空间。
创建BFC:
根元素
display的值是inline-block flex或者inline-flex
position的值不是static或者relative
overflow:hidden
BFC的特性：
1内部的Box会在垂直方向上一个接一个的放置
2垂直方向上的距离由margin决定
3 bfc的区域不会与float的元素区域重叠
4 计算bfc的高度时 浮动元素也参与计算
