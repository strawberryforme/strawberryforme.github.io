    2.prop

    场景：子组件接受 外部组件传入的值
1.传递数据 
    子组件+ 属性名 + 值。(属性名不能是key，ref)
    子组件+ v-bind + 属性名 + 变量。(v-bind: ""引号里面的值，表达式执行的结果。传过去。)

    1.传入一个字符串：
    <子组件 sex="man">标签属性. 静态传入一个字符串。如果传入18，那么sex=18.
    <子组件 :sex="user.sex">标签属性.动态赋予一个变量的值。变量的值会变为字符串。 
    
    2.传入一个数字：
    <blog-post :likes="42"></blog-post> //即便 `42` 是静态的，我们仍然需要 `v-bind` 
    <blog-post :likes="post.likes"></blog-post> 

    3.传入一个布尔值：
    <blog-post :is-published="false"></blog-post>
    <blog-post :is-published="post.isPublished"></blog-post>

    4.传递数组：
    <blog-post :comment-ids="[234, 266, 273]"></blog-post>
    <blog-post :comment-ids="post.commentIds"></blog-post>

    5.传入对象：
    <blog-post :author="{name: 'Veronica',company: 'Veridian Dynamics'}"></blog-post>
    <blog-post :author="post.author"></blog-post>
    传入对象所有property：
    <blog-post v-bind="post"></blog-post>
    <blog-post :id="post.id" :title="post.title"></blog-post>
    
    6.也可以使其他构造函数。

    2.computed

    场景：1个数据由多个数据决定。则比如购物车。
原理：底层借助了Object.defineproperty方法提供的getter和setter
优点：1、将属性值计算专门存放。代码从模板分离。2、如果没有发生改变，会从缓存中读取。
缺点：计算属性靠返回值返回，所以不能处理异步方法，


定义：通过计算而得来的动态属性, 该计算结果会被缓存起来。
调用：computed内部的函数在调用时不加()。 ,in" "
触发：1、模板第一次渲染的时候。2、关联属性修改的时候。

写法1：必须要return。fullName:{get() {if(){return } else return}} 。
写法2：关闭缓存，默认为true 。cache:false。return Data.now() + this.text
写法3：如果自己有修改，必须set。fullName:{get() {return},set(value){}//} this.didiFamily = 'John Doe'。
写法1简写： fullName{get(){}}。allname:这个就是fullName:get(){},===function(){}。allname(){}//简写。是个属性不是函数哦。
语法限制：必须是data和prop没有出现过的属性，data属性从而触发computed中的函数，形成死循环了。函数必须要有return。

    3.watch

    场景：一个数据影响多个数据.当需要在数据变化时执行异步或开销较大的操作时.
原理： watch深层监听数据的变化。
优点： 相当于一个观察者模式。监听数据改变，进行同步或者异步操作。
缺点： watch没有缓存。

定义： watch是一个key，value形式。可以监听data，prop，computed，路由
调用： 1、数据第一次加载不会调用，数据改变时候，会被动调用。2、如果数据第一次加载需要调用，用immdeate。
属性和函数： 1.immter。2.deep。3.handler(写其他函数没有用)。

写法1：Vue完整写法，1函数2属性：watch:{data1{ handlder(new,old){}; immediate:true;deep:true}}
写法2：Vue只写1函数的时候，简写watch:{data1(new，old){}}
写法3：实例写法：vm.$watch('data1',{handler(){};immediate:true;deep:true})
写法4：实例写法，简写：vm.$watch('data1',function(new,old){})

语法限制：
1、检测的可以是data，可以是props，可以是computed.
2、深度监听，不能直接写number.a不能成为key因为是字符串
3、Vue里面的函数不要用箭头函数。this需要绑定Vue实例的。

    1、条件渲染v-if:
    写法：v-if="表达式"，v-else-if="",v-else。

    适用于：切换频率低的场景。
    注意：v-if可以单数使用。也可以和v-else-if，v-else共同使用时不能有断开。
    配合：可以和template配合使用。<template v-if=""></template>
2、条件渲染v-show:
    写法：v-show="表达式"
    原理:通过改变display:none和displaye:block改变        
    适用于:切换频率高的场景。
    特点：不展示DOM元素未被移除，仅仅使用样式隐藏。元素都可以获取。

    3、key原理

    一、有key和无key对比
    需要知道的知识点：
    1、

    有key：

    数据的渲染更新：初始数据-》虚拟DOM（+diff算法。）-》真实dom。
    
    两次虚拟DOM有一个DIFF算法。
    (1)时虚拟DOM中未找到与新虚拟DOM相同的key
    创建新的真实DOM，随后渲染到到页面。
    （2）找到了，
    1、虚拟DOM内容没有变，直接用。
    2、虚拟DOM内容变了，生产的新的真实dom值。
     
    没有key：默认是有key。key=index。就地复用，
    
    二、实例。
    如果key=index
    li key = 0  li key=0
    li key = 1  li key = 1
    li key = 2  li key =2
                li key = 3
    虚拟dom的文字节点：是不一样的。那么就更换。
    虚拟dom的input输入框节点：是一样的。 那么就不更换。
    更换后的 +++ 遇见了 没有更换的 节点框拉胯了。
    
    key=id
    key==001 ,key=004(重新生成新的。)
    key=002,  key =001,
    key=003,   key=002,
                key =003
    
    明显看出，当有渲染数据文本节点+input节点的时候。
    1、对于数据进行：逆序添加，逆序删除 破坏操作。
    会产生没有必要的真实dom更新===》页面效果没有问题，但是效率低。
    可以提高效率，复用其他的。

    2、如果结构中有输入类的dom：
    会产生错误dom更新=》界面有问题。
    防止 文本 和 input错位。
    
    三、总结：
    1、为什么要有key。key作用：
    1、key的作用主要是为了高效的更新虚拟DOM。且不产生错误的DOM更新。
    2、另外vue中在使用相同标签名元素的过渡切换时，也会使用到key属性，其目的也是为了让vue可以区分它们，否则vue只会替换其内部属性而不会触发过渡效果。

    2、开发使用key：
    1、唯一表示。id，身份证号，手机号。
    2、如果不存在数据逆序添加，逆序虚拟删除破坏操作。用index为key是可以的。
