    1.基本概念

    # 1.基本概念

    名词：
    Vuerouter：是一个插件库，专门实现SPA应用。
    SPA： 只有1个完整页面index.html。点击导航链接不会刷新页面，局部刷新，
    router: 包含1个routes
    routes: 包含多个route
    route: 路径 path:'/' =》Component:''
种类：
    Vuerouter有两种。
    hash模式：
        特点：对于一个url，has值。就是#后面的内容。
        优点：1、兼容性好。2、hash值不会包含http请求。不会带给服务器。
        缺点：1、带着#号键不美观。2、app分享，地址因为#标记不合法。
    history模式：
        优点：1、地址没有#美观。
        缺点：1、兼容性较差。2、应用部署上线后需要后端支持，解决页面服务端404问题。
原理：
    1.切换原理：通过切换组件，隐藏的路由组件是会被销毁的，需要的时候重新挂载。
    2.每个组件都有$route属性，存储自己的路由信息
    3.整个应用只有一个router，可以使用组件的$router属性获取。

    # 2.创建和使用

    一、创建路由router：pages.index.js:

    1.引入Vuerouter库、组件。
    2.创建router: const router =new Router(routes)。router有1个routes。
    router属性：
        mode: "history",
        base: process.env.BASE_URL,
    3.创建routes：const routers = [{}]。
    4.route的属性：
        核心路径:
            path:'', // 一级路由：path需要加/ 多级路由：path不需要加/
        核心组件名:
            component:Component //不需要加path
        组件名称:
            name: 'hello'。
        路由元信息:
            meta:
        重定向: 
            redirect: '/'
            redirect: { name: 'homepage' }
            redirect:to => { return { path: '/search', query: { q: to.params.searchText } } },
            redirect: to => { return 'profile'}
        别名：
            alias: '/home'
            alias: ['/people', 'list']
    5.导出：export default router

    二、挂载router：main.js

        导入VueRouter和router：import VueRouter from 'vue-router'，import router from './router'
    Vue全局使用Vuerouter库：Vue.use(Vue.router)
    Vue实例挂载router路由：Vue({ router:router })

    三、路由跳转

    一、声明式导航
1.组件router-link创建链接。
    属性1.to.
        to:相当于href，后面跟的是组件。<a href= "www.baidu.com"/>
        1、to+路径: 
            路由声明一级: path:'/home'
            路由声明二级：children:{path:'message'} //不能写/message
            跳转全写：:to="{path:"/home"}", 
            跳转简写：:to="/home/message"。
        2、to+名字:
            路由声明: name:'hello'
            跳转字符串写法: :to="{name:'hello'}"。
            跳转对象写法：:to="{name:'title'}"
        3、to+query参数：
            3.1、to的字符串写法
            跳转全写：router-link :to="`home?id=${m.id}&title=${m.title}`"
            接受参数：$route.query.id
            3.2、to对象写法
            跳转全写：router-link :to="{ path:'/home',query:{id:m.id, title:m.title}}
            接受参数：$route.query.id
        4、to+params参数：
            4、1to的字符串写法：
            路由声明： path:'message/:id/:title' 需要有占位符
            跳转传递： router-link :to="`/home/message/666/你好`" 
            接受参数：  
            4、2to的对象写法:不允许使用path，必须使用name。
            路由声明： path:'message/:id/:title' 需要有占位符
            跳转传递：router-link :to="{ name:'',params:{id:m.id,title:m.title}}"
            接受参数：  
        5、路由的prop配置：(简化路由接受参数)
            5.1 prop值为对象：
            该对象中的所有key-value都会以prop传给改路由
            Detail路由声明：{path:'/detail/:id/:title',component:'Detail',props:{a:1,b:'hello'} }
            Detail组件接受：props:['a','b']
            5.2 prop值为布尔：
            若布尔值为真，会吧该路由组件接受到所有params参数，以props传给Detail组件
            Detail路由声明：{path:'/detail',component:'Detail',props:true }
            Detail组件接受：prop:['id','title']
            5.3prop值为函数
            props($route):{return {id:$route.query.id,title:$route.query}}
            props({query}):{return {id:query.id,title:query}}
    
    属性2.激活属性
        active-class="active"

    属性3.replace属性
        浏览器的历史记录有2种。
            push：追加历史记录，默认为push。
            repalce：替换当前记录。
        语法：
            全写:replace='true'
            简写：repalce
    
2.显示url对应的组件 router-view
    将显示与 url 对应的组件。你可以把它放在任何地方，以适应你的布局。
    
    这里说明一下：跳转和嵌套没有直接关系。
    更新一下意识，局部更新的内容，用组件跳转。组件显示用div。

    router-link 就是链接。 route-view 是链接后的展示。
    A组件内部，router-link 跳转到B组件。B组件在下方显示router-view。或者在A组件上方显示。
    B组件内部，router-link 点击传入参数1,2,3转到C组件。B组件传什么。C组件展示什么。 多一个C需要接受。
    
    路由嵌套呢。只是使用路径的嵌套。按照显示来说。因为是vuerouter局部的，在哪里都可以显示。（和父子组件没有直接关系）
    
    说明一下：和a标签的区别。
    之前是写 a标签，会进行跳转。会跳转到新页面。或者是页面上下。
    现在是写router-link，点击后，进行路径切换 和 其他组件的显示。
    其他组件想显示在哪就显示在哪。     

二、编程式导航

        },

        },
    }

    # 4.路由守卫

    1、全局前置路由守卫:
    作用：
    对路由权限的控制。初始化时候被调用，每次路由切换之前调用
    语法：
        router.beforeEach( (to,from,next) => { console.log('@') } )
    其中，
        第一个参数to：是要去的路由。有路径，hash，原信息，参数。
        第二个参数from：是要来的路由。有路径，hash，原信息，参数。
        第三个next(拦截)：是进行下一步。否则就要停住。
    例子：
        我们不会对所有的路由进行限制。我们只会要去路由的原信息进行校验，如果是真。那么通过。
        只需要在需要权限的路由上加上 meta:isAuth:flase。
        如果需要路由需要鉴权，那么进行判断。通过的话next展示，不通过不显示next。
        如果当前路由没有鉴权，直接next展示无需权限的路由。

2、全局后置守卫
    作用：
        对路由权限的控制。初始化时候被调用，每次路由切换之前调用
    语法：
        router.afterEach((to,from)=>{})
    例子：
        meta:{title:'详情'}
        document.title = to.meta.title || 'caomei'
        document.title = to.meta.title 修改网页的title
3、全局解析守卫：
    语法：router.beforeResolve( async to=>{})  

4、路由独享守卫
    作用：
        对route内的当前路由进行判断。可以和全局后置路由配合使用。
    语法：
        route:[{ path:'/about',component:A,beforeEnter({to,from,next}) }]
5、组件内守卫
    直接在当前组件里面：
    通过路由规则，进入这个路由：beforeRouteEnter( to, from ,next){ n()}
    进入这个路由：beforeRouteUpdate( to, from ,next){}
    通过路由规则，离开这个路由：beforeRouteLeave( to, from ,next){ n()}
    
    1、home-> about 前置路由守卫，后置路由守卫：调用两次。
    2、home- > about ->text ：组件内守卫。每一次动作调用1次beforeRouteEnter -> beforeRouteLeave
